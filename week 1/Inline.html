Inline elements

Why do we bother with inline elements? There are certainly times we want to mark up content, but we don´t want to insert the line breaks around them. The tag was the first to give you that ability, but it was mostly used to style the code, and we don't want to do that. Some of the newer tags that still separate content and style are listed below. Remember, inline tags can flow next to other and don´t cause a line break before and after the element. We haveseen this with links and images.

Defining your abbreviations - abbr

The abbr will display an abbreviation on the browser. But if a title attribute is specified in the tag, the title will appear when the mouse hovers over the abbreviation. Screen readers also provide the title information.

The days of the week are often abbreviation: Sun, Mon, Tues, etc.

Specifying the use of time - time

The time is again a completely semantic tag that can be used to search for specific information.

This party starts at 9:00pm.

The party took place on New Year's Eve.

Identifying code fragments - code

The code elements tends to be displayed in a different font, but also has the semantic meaning to convey information to as many people as possible
